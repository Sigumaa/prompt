しゆい、じゃあ「これ解く→これで勉強→次これ→もう一回解く」だけに絞って、公式だけで回せる順番をそのまま置くね。

まず最初は、いきなり公式のPractice Assessmentを1回だけ解いてみる。ここは点数を気にしないで、「どんな聞かれ方をするか」と「自分が落とす領域」を炙り出すための診断。終わったら、間違えた問題の領域（生成AI／NLP／CV／ML基礎／AIワークロード）だけメモして次へ。([Microsoft Learn][1])

```text
https://learn.microsoft.com/ja-jp/credentials/certifications/exams/ai-900/practice/assessment?assessment-type=practice&assessmentId=26
```

次に、そのままMicrosoft Learnのラーニングパスを上から最後まで通して勉強する。ここは「一周して全体像を作る」が勝ちで、途中で完璧主義にならないほうが速い。終わった判定は、“このユースケースならこの系統のサービス”が雑でも言えること（例：画像→Vision、テキスト→Language、音声→Speech、生成→生成AI）。([Microsoft Learn][2])

```text
https://learn.microsoft.com/ja-jp/training/paths/introduction-to-ai-on-azure/
```

次に、学習ガイド（Study guide）を開いて、さっきメモした「落とす領域」が学習ガイド上でどこに書かれてるかを確認して、そこだけLearnに戻って該当モジュールを復習する。終わった判定は、間違えた問題を自分の言葉で説明し直せることと、「なぜその選択肢が違うか」を言えること。([Microsoft Learn][3])

```text
https://learn.microsoft.com/ja-jp/credentials/certifications/resources/study-guides/ai-900
```

それが終わったら、もう一回Practice Assessmentを解いてみる。ここで「さっき落とした領域が上がったか」だけを見る。まだ落ちるなら、また学習ガイド→該当モジュール復習→Practiceをもう一往復、でいい。([Microsoft Learn][1])

```text
https://learn.microsoft.com/ja-jp/credentials/certifications/exams/ai-900/practice/assessment?assessment-type=practice&assessmentId=26
```

最後に、Exam sandboxを触って試験画面に慣れて終わり。知識があっても操作で迷うのが一番もったいないから、ここだけは短くても入れるのが綺麗。([Microsoft][4])

```text
https://aka.ms/examdemo
```

もし「文章で学ぶより、講座形式で一気に掴みたい」タイプなら、上のラーニングパスの代わりに（または並行で）公式コースAI-900T00を使うのもアリ。狙いがAI-900と同じ（AIワークロードの認知と、支えるAzureサービスを特定できること）って明言されてる。([Microsoft Learn][5])

```text
https://learn.microsoft.com/ja-jp/training/courses/ai-900t00
```

この手順で回すと、「解く→弱点が見える→そこだけ学ぶ→もう一回解いて潰す」が一直線になるから、寄り道せずに合格ラインに乗せられるよ。
